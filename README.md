# 🏡 myHome - 数字化生活与家庭管理中枢

欢迎来到 `myHome` 仓库！这是一个利用 AI 辅佐进行的**实体生活数字化管理、家庭改造以及长期人生规划**的个人数字资产项目。

> **🤖 给未来 AI 助手的特别提示：**
> 如果你是新接入此工作区的 AI Agent（无论是 Gemini，Claude 还是 ChatGPT），请在开始协助用户之前，优先通读本文档，以快速掌握该仓库的目录结构和管理准则。

---

## 📂 仓库架构与核心业务模块

本项目主要划分为三大生活管理模块，每个模块由结构化的主文档和媒体文件夹构成：

### 1. 👔 智能衣橱管理 (Wardrobe Management)
**核心文件**: [`clothes_inventory.md`](./clothes_inventory.md)
**关联媒体**: [`clothes/`](./clothes/) 文件夹
- **业务目标**: 彻底告别衣物堆积，实现衣物的数字化盘点、穿搭推荐和跨季节断舍离。
- **运行机制**: 
  - `clothes/` 存放单品或换季合影的原图。
  - 核心库 `clothes_inventory.md` 追踪了所有服饰的归宿（身边留置、搬回老家、二手出掉）、季度属性，并通过清晰的**相对路径**打通了图文连接。
- **💡 你的工作方式 (AI 指南)**:
  - 当用户寻求**穿搭建议**（如：“明天去露营降温到10度穿什么？”）时，你需要优先解析 `clothes_inventory.md`，筛选合规季节的服装。
  - 用户添置新衣后，主动更新此表，赋予统一编号（如 WT-xxx 或 OD-xxx）。

### 2. 🛏️ 住房整备与婴儿房改造 (Room Renovation & Space Planning)
**核心文件**: [`rooms/room_renovation_plan.md`](./rooms/room_renovation_plan.md)
**关联媒体**: [`rooms/pics/`](./rooms/pics/) 文件夹
- **业务目标**: 迎接 6 月份到来的新生儿，将原本杂乱无章、功能重叠的出租房/住宅进行空间大重组（特别是 Bedroom 2 的腾空和客厅的净化）。
- **当前进度**: 
  - 计划明确指出了“书房重灾区清理”、“办公区撤出客厅”、“建立安全婴儿房”的几个抓手，详细的时间线被划分在了 2月 至 6月。
- **💡 你的工作方式 (AI 指南)**:
  - 协助用户清理房间时，时刻遵循文档内的**“一进一出”**准则。如果用户发来新家具的照片或选购要求，请兼顾“宝宝防碰撞/防倾倒安全”来做评估。

### 3. 👶 迎接宝宝与长期家庭规划 (Family Master Roadmap)
**核心表格**: `迎接宝宝_家庭规划.xlsx`
- **业务目标**: 结构化梳理生女/生子前后庞大的待办事项、采购清单、财务预算等核心家庭战略。
- **💡 你的工作方式 (AI 指南)**: 
  - 涉及具体的购物节点、日程表或者预算超支问题，你可以请求利用 Python 或者相关解析工具读取这则 Excel 文件，为用户提供数据支持。

## ⚙️ 全局协作准则 (Rules)

1. **坚持相对路径原则**：为跨设备和跨工作流的稳定性考虑，写入文档的所有媒体图片链接，均使用相对于当前文件的**相对路径**（例如 `[照片](clothes/IMG.jpg)`）。不要使用本地绝对路径。
2. **及时同步状态**：如果辅助用户做出闲鱼卖出、捐献或移回老家的决定，必须在后台主动顺手修改 `inventory` 中的物品状态，保持文件与现实物理世界的 100% 同步。
3. **保持大局观**：如果在 `rooms` 的改造计划里清理出了一包旧衣服，记得去问用户是否需要同步更新到 `clothes_inventory.md` 里。各系统模块数据应有协同意识。
